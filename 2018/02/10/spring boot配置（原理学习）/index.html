spring boot配置（原理学习）

发表于 2018-02-10 | | Views

注：本文中配置和代码分析基于当前spring boot最新版1.5.x

概述

写了较长时间的spring boot项目，对于其中的原理一直没有很好的看过，以至于每次看那些注解，都不知道为什么要标注。就配置而言，什么时候该标注@Configuartion，什么时候该标注@EnableAutoConfiguartion，他们到底为我们做了什么，这些是不是我们需要的，还是会引起其他的问题。所以这次也是希望能够简单走读下spring boot的代码，了解一些基本的原理，对于使用也是有好处的。

本文旨在分析spring boot配置相关的注解，后续不定期更新其他部分或者其他项目的分析~

@EnableAutoConfiguartion

原理参考：https://www.jianshu.com/p/2a171fa80756

EnableAutoConfiguration是一个接口，通过@Import(AutoConfigurationImportSelector.class)，代理AutoConfigurationImportSelector实现逻辑

AutoConfigurationImportSelector类实现了DeferredImportSelector接口，主要是在selectImports引进需要的配置，跟进代码，发现是读取META-INF/spring.factories文件，这个文件中列了很多spring boot预定义的配置类，以下用其中的ActiveMQAutoConfiguration类来分析

首先参考spring boot和activemq结合的使用例子（参考：http://blog.csdn.net/liuchuanhong1/article/details/54603546）

可以看到配置项都是spring.activemq开头的配置

其次看ActiveMQAutoConfiguration类中的@ConditionalOnClass({ ConnectionFactory.class, ActiveMQConnectionFactory.class })，也就是在classpath检测到这些类（引入了相关的jar包）的情况下才进行当前类的配置

然后看到ActiveMQAutoConfiguration类中的@Import({ ActiveMQXAConnectionFactoryConfiguration.class, ActiveMQConnectionFactoryConfiguration.class })，可以继续跟进ActiveMQConnectionFactoryConfiguration看看究竟是怎么配置的

跟进ActiveMQProperties类，发现了@ConfigurationProperties(prefix = “spring.activemq”)注解，从这里看，应该就是spring boot去读配置文件中前缀是spring.activemq的配置项，包括spring.activemq.broker-url等，注入到当前ActiveMQProperties类的实例中，用于activemq工厂类的初始化

上述过程中，spring boot实际上是暴露了一些配置项，然后通过@EnableAutoConfiguartion注解标识，自动的为我们创建好通用的配置bean，降低spring boot和其他框架的集成成本

从上面spring的实现思路来看，spring保留了很好的扩展性，扩展性在于我们可以定义自己的注解（类似于ConditionalOnProperty），只要符合spring的要求，给我们的注解再注解以@Conditional，并指定我们自己的Condition实现类即可。spring通过对java注解的追踪，可以为上述过程提供保障。
